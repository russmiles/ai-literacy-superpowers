# Governance Dashboard Specification

A self-contained HTML file following the portfolio dashboard pattern.
Single file, no external dependencies, works offline. Generated by
`/governance-health --dashboard` or as part of `/governance-audit`
output.

## Sections

### 1. Governance Health Summary (top of page)

**Layout:** Three metric cards in a row.

| Card | Value | Colour logic |
| --- | --- | --- |
| Overall Score | Composite of falsifiability ratio, drift score, debt count | Green > 0.75, amber 0.50-0.75, red < 0.50 |
| Constraint Quality | Count by tier: falsifiable / partial / vague | Stacked bar in green/amber/red |
| Last Audit | Date with staleness indicator | Green < 90d, amber 90-180d, red > 180d |

### 2. Constraint Quality Table

**Layout:** Sortable table.

| Column | Source |
| --- | --- |
| Constraint name | HARNESS.md governance constraint heading |
| Falsifiability | Audit report: falsifiable / partial / vague |
| Drift risk | Audit report: low / medium / high |
| Drift stage | Audit report: 1-5 (see drift-stages.md) |
| Frame alignment | Audit report: aligned / divergent / unchecked |
| Promotion level | HARNESS.md: unverified / agent / deterministic |
| Last reviewed | Audit report date or "never" |

### 3. Governance Debt Inventory

**Layout:** Table sorted by score descending.

| Column | Source |
| --- | --- |
| Description | Audit report debt item |
| Severity | 1-3 scale |
| Blast radius | 1-3 scale |
| Score | severity x blast radius |
| Affected constraints | Comma-separated list |
| Recommended action | Audit report recommendation |
| Age | Days since first detected |

### 4. Drift Timeline

**Layout:** Line chart (CSS-drawn, no JS library dependency).

- X axis: audit dates (quarterly data points)
- Y axis: drift score (mapped to 0-1 scale: low=0.2, medium=0.5,
  high=0.8)
- Line colour: green when decreasing, red when increasing, grey
  when stable
- Overlaid: falsifiability ratio as a second line

### 5. Three-Frame Alignment Heatmap

**Layout:** Grid with one row per governance constraint.

| Column | Colour |
| --- | --- |
| Engineering | Green = aligned, amber = partial, red = divergent, grey = unchecked |
| Compliance | Same scheme |
| AI System | Same scheme |

### 6. Trend Comparison (if multiple audits exist)

**Layout:** Sparkline-style mini charts.

| Metric | Chart type |
| --- | --- |
| Falsifiability ratio | Line trending up/down |
| Debt inventory size | Line trending up/down |
| Drift velocity | Arrow indicator (stable/increasing/decreasing) |

## Data Source

The dashboard reads JSON data embedded in a `<script>` tag within
the HTML file. The governance-auditor agent or `/governance-health`
command serialises audit report data into this format when generating
the dashboard.

## Colour Scheme

Follow the existing portfolio dashboard colours:

- Green: `#22c55e` (healthy)
- Amber: `#f59e0b` (needs attention)
- Red: `#ef4444` (critical)
- Grey: `#6b7280` (unchecked / no data)
- Background: `#1a1a2e` (dark theme)
- Text: `#e2e8f0` (light on dark)
